# main-home-archive

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能包含静态内容、小型工具、信息展示或实验性项目。仓库不绑定任何特定域名或网站，仅作为页面文件的集中存放与版本管理。

## 目录说明

- `pages/` — 存放所有独立的 HTML 页面文件，每个页面可自包含样式与脚本。
- `assets/` — 公共资源文件，如图片、字体、CSS 或 JavaScript 库。
- `docs/` — 与页面相关的说明文档或使用指南（可选）。
- `archive/` — 历史版本或不再维护的页面备份。

## 页面归档说明

每个页面均以独立文件形式存在，文件名应反映其用途或内容主题。如需引用页面，可直接使用文件路径或链接到 GitHub Pages 部署后的对应地址。页面之间无依赖关系，可单独访问。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有公共资源请放入 `assets/`，并更新本 README 的“页面列表”部分（可选）。
- 更新页面：直接修改对应文件，建议在提交信息中注明变更内容。
- 删除页面：将文件移入 `archive/` 目录，避免永久删除影响历史引用。
- 欢迎提交 Pull Request 改进现有页面或添加新内容，但请确保页面不包含恶意代码、跟踪脚本或外部依赖（除非必要）。
